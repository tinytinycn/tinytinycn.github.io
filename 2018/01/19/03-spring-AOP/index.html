AOP

  • 面向切面编程, 通过预编译方式或运行时动态代理方式, 实现业务的扩展.
  • 面向切面编程, OOP面向对象编程的补充和完善.
  • AOP把系统分为两部分, 核心关注点和横切关注点; 核心主要处理业务逻辑, 横切主要是权限验证/日志/事务处理等.

Spring AOP

  • Spring AOP 框架核心模块之一.
  • Spring AOP 依托 @AspectJ框架, 底层通过动态代理方式实现AOP.
  • Spring AOP 应用特点: 基于OCP开闭原则, 在不改变原有功能基础上扩展新功能. 实现业务代码解耦, 提高代码复用性, 简化代码编写, 提高效率.
  • Spring AOP 应用场景: 将软件业务划分为核心业务和非核心业务(或核心关注点和横切关注点). 如: 系统日志处理, 系统安全验证, 系统事务处理, 系统缓存处理等.

Spring AOP 原理

Spring AOP 底层依赖于代理机制, 在对象运行时动态注入新扩展的功能.(动态代理参见: 代理模式.md)

  • 切面Aspect: 横切面对象(可使用注解@Aspect声明)
  • 连接点JoinPoint: 程序执行过程的特定点, 一般指拦截到的方法
  • 切入点PointCut: 对连接点拦截内容的定义
  • 通知Advice: 在切面的某个连接点上执行的动作

注解实现方式:

  1. 创建maven java项目, 添加依赖 (同上)
  2. 创建核心业务接口和实现类
  • 实现类添加@Service注解
  1. 创建扩展业务类(切面类), 定义切入点和增强处理 (同上)
  • 切面类添加@Component注解和@Aspect注解
  • 切面类扩展方法添加@Before/@After/..等增强通知注解
  1. 添加spring配置文件
  • 开启自动扫描和注解即可
  1. 创建测试类
