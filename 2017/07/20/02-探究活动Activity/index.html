02_探究活动Activity

Jul 20, 2017

1.Activity 和 当前界面

2.创建活动Activity / 创建和加载布局 （逻辑和视图分离 设计）

  • setContentView()方法来给当前的活动加载一个布局。
  • 所有的活动都要在AndroidManifest.xml中进行注册才能生效。（Android studio自动完成注册！）

3.活动Activity中使用Menu菜单

  • res>New>Directory 新建main.xml 一个menu文件
  • 活动Activity类中，重写 onCreateOptionMenu() 方法
  • 在上方法中，通过getMenuInflater() 方法得到MenuInflater 对象，再通过inflate() 方法可以创建活动菜单。
  • 活动Actiivty类中，重写 onOptionItemSelected() 方法

4.销毁活动

  • finish() / Back键

5.Internt 显示和隐式使用

  • 指定目标活动，意图明显，称为显示Intent。
  • AndroidManifest.xml中通过指定一系列抽象的action 和 category信息，交由系统分析找出合适的活动去启动，称为隐式Intent。
  • 中可以配置标签，精确地指定响应什么类型的数据。

6.传递数据给 下一个活动Activity

  • Intent中提供一系列 putExtra()方法重载

7.返回数据给 上一个活动Activity

  • start ActivityForResult(intent,200)方法来启动下一个活动
  • 回调上一个活动的 onActivityResult(int requestCode, int resultCode, Intent dataIntent)方法

8.Activity 的生命周期

  • Android是通过任务Task来管理Activity的，一个任务就是一组存放在栈里的活动集合
  • 返回栈Back Stack（后进先出 的原则）
  • 活动状态：运行、暂停、停止、销毁 4种状态
  • 回调方法：onCreate() onStart() onResume() onPause() onStop() onDestory() 和 onRestart() 7个方法
  • Ctrl + O （重写方法自动补全小技巧）
  • onSaveInstanceState()回调方法，解决临时数据保存问题
  • onCreate(Bundle savedInstanceState)中，savedInstanceState 这个Bundle对象通过相应取值方法将保存数据取出

9.Activity 的启动模式

  • 一个活动的启动模式：standard、 singleTop、 singleTask、 singleInstance 4种
  • 标签指定 android:launchMode属性
  • standard模式下，每当启动一个活动时，就会在返回栈中入栈，并处于栈顶。（系统不会在乎此活动是否已经在栈中存在，每次启动都会创建一个活动的新实例）
  • singleTop模式下，启动时，系统发现返回栈的栈顶已经是该活动，则直接使用它，不再创建一个活动的实例。（当要启动的活动不处于栈顶，则会创建一个活动的新实例）
  • singleTask模式下，启动是，系统检查返回栈中是否已经存在该活动实例，存在则直接使用它（并把这该活动之上的所有活动统统出栈）。不存在则创建一个活动的新实例。
  • singleInstance模式下，会启动一个新的返回栈来管理要启动的这个活动。不管哪个应用程序App来访问这个活动，都公用的同一个返回栈，解决了共享活动。
