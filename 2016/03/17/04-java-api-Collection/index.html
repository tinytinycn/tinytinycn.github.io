Collection集合

Collection 是一个接口，定义了集合相关的操作方法。两个子接口:

  • List 可重复集合
  • Set 不可重复集合

注意：集合中存储的都是 引用类型 元素，集合只保存每个元素对象的引用，并非元素对象本身。

集合常用方法:

  • add(Object o)
  • contains(Object o) 集合是否包含该元素，根据每个元素的equals()方法进行比较的结果进行判断
  • size() 不同于数组 array.length; 不同于字符串 s.length();
  • clear(); 集合移除所有元素
  • isEmpty(); 集合是否为空
  • addAll(Collection c)添加集合
  • containsAll(Collection c) 集合是否包含指定集合

泛型

Java 5 加入泛型机制。泛型的本质，就是参数化类型。

注意：泛型机制在运行期间时没有的，只在编译期存在。

Iterator迭代器

Iterator是一个接口，集合重写Collection的iterator()方法，利用内部类提供提供了迭代器的实现。
方法：

  • hasNext() 是否有下个元素
  • next() 获取下一个元素
  • remove() 迭代器的remove操作移除元素， 不能在遍历时使用集合的remove()方法移除元素，否则异常。

forEach循环

Java 5 加入增强for循环。用于遍历集合或数组。
