面向对象

  • 类(型): 模板,模型
  • 对象: 创建出的实例(存在于内存中)
  • “一切皆是对象”

对象的创建

实例化: 创建对象的过程,创建出来的对象叫做实例.

对象的使用

引用对象: 声明一个引用变量来引用对象,通过引用变量访问对象的属性(数据,方法)

方法

方法签名: 方法名+参数列表 (作为方法的唯一标识)
方法重载: overload (两同一不同: 同类同名不同参)

构造方法

类的成员,可以看做特殊的方法.(构造函数/构造器)

注意:

  • 构造方法没有返回值.
  • 任何一个类都有构造方法, 默认生成无参构造方法.
  • 一个类中,只有一个无参构造方法.
  • 一个类中,可以有多个有参构造方法.
  • 继承中,构造方法的调用: 创建子类对象,可以通过super调用父类构造方法(super()必须位于构造器第一行)
  • 继承中,子类(无参/有参)构造方法默认都会调用父类无参构造方法.
  • 继承中,子类有参构造方法会覆盖默认无参构造方法,除非显式写出无参构造方法.
  • 继承中,子类有参构造方法默认调用父类无参构造方法,若父类没有无参构造方法则报错.(可以显式super调用父类有参构造方法)

this

  • 表示当前类的对象,一般省略.
  • 有参构造器中不可以省略.
  • 方法中使用, 指向调用该方法的对象.

JVM 内存管理

Java数据都需要存储在内存中.JVM内存分为 方法区,堆,栈,各自存储不同的数据.

  • 栈: 当执行一个方法时,Java内存会在(虚拟机)栈中开辟一块区域(栈帧),存放这个方法相关的局部变量表,操作数,动态链接等等.
  • 堆: 保存new 出来的实例对象.(成员变量保存在堆中)
  • 方法区: 保存类Class的结构信息包括静态成员和方法.(类加载器载入类文件字节码信息,解析后放入方法区.) Java8 方法区存储位置发生细微变化*

内存泄露和内存溢出

  • 内存泄露: 分配出去的内存回收不回来.
  • 内存溢出: 内存泄露到一定程度,导致溢出.OutOfMemoryException.
